# 記事作成プロジェクト

## プロジェクト概要

本プロジェクトでは、AI エージェント（Reviewer と Writer）を活用して、技術記事を作成します。10回のイテレーションを通じて記事の品質を段階的に向上させることを目指します。

## エージェント構成

### Reviewer エージェント

- **役割**: スタイルガイド作成、記事レビュー、品質評価
- **責任範囲**:
  - 情報収集とスタイルガイド策定
  - Writer の作成した記事のレビューと評価
  - レビュー結果に基づくスタイルガイドの更新

### Writer エージェント

- **役割**: 記事執筆
- **責任範囲**:
  - Reviewer が作成したスタイルガイドに基づく記事執筆
  - フィードバックに基づく記事の改善

## プロセスフロー

### Phase 1: 初期セットアップ

1. **情報収集** (Reviewer)

   - 基本情報収集
   - 公式ドキュメント、GitHub、関連記事の調査
   - 技術的特徴、使用例、ベストプラクティスの整理

2. **スタイルガイド作成** (Reviewer)
   - 収集した情報に基づく初期スタイルガイド作成
   - 記事の構成、トーン、技術的深度の定義

### Phase 2: イテレーティブ改善 (10回)

各イテレーション（2〜10回目）で以下を実行：

#### ステップ 1: 記事作成 (Writer)

- 現在のスタイルガイドに基づく記事執筆
- 前回のフィードバック反映

#### ステップ 2: レビューと評価 (Reviewer)

- 記事の品質評価（10点満点）
- 詳細なフィードバック提供
- スタイルガイドの更新

### Phase 3: AIということの開示

記事の最後のセクションにAIでの記事作成を試している旨を記載する

## レビュー基準

参考記事：[AI が技術記事を書く時代のレビューについて](https://zenn.dev/uhyo/articles/ai-writes-tech-articles-202510)

### 評価項目（各2点満点、計10点）

#### 1. 技術的正確性 (2点)

- **2点**: 技術的な内容が完全に正確で、実装例や説明に誤りがない
- **1.5点**: ほぼ正確だが、細かい技術的な不正確さがある
- **1点**: 一部に明確な技術的誤りがある
- **0.5点**: 複数の技術的誤りがある
- **0点**: 技術的内容が大幅に間違っている

#### 2. 自然な文章フロー (2点)

- **2点**: 文章が自然で読みやすく、論理的な流れがある
- **1.5点**: 概ね自然だが、一部ぎこちない表現がある
- **1点**: 文章フローに改善の余地がある
- **0.5点**: 不自然な表現や論理の飛躍が目立つ
- **0点**: 文章として成立していない部分がある

#### 3. 記事構成・ナラティブ (2点)

- **2点**: 読者にとって分かりやすい構成で、物語性がある
- **1.5点**: 良い構成だが、一部改善点がある
- **1点**: 構成は理解できるが、魅力に欠ける
- **0.5点**: 構成が分かりにくい
- **0点**: 構成が破綻している

#### 4. 実践的価値 (2点)

- **2点**: 読者が実際に活用できる具体的な情報が豊富
- **1.5点**: 実践的だが、もう少し具体性があると良い
- **1点**: 実践的価値はあるが限定的
- **0.5点**: 実践的価値が低い
- **0点**: 実践的価値がほとんどない

#### 5. 人間らしさ・個性 (2点)

- **2点**: 個人的な経験や見解が含まれ、人間らしい「不完全さ」がある
- **1.5点**: 人間らしさがあるが、やや型にはまっている
- **1点**: 一部に個性があるが、全体的に定型的
- **0.5点**: ほとんど定型的で個性がない
- **0点**: 完全に機械的で個性が全くない

### 目標スコア

- **最終目標**: 8.5点以上（合格基準）
- **優秀**: 9.0点以上

## 基礎情報

### 基本概要

- **定義**: Web標準に基づいて構築されたWebアプリケーションフレームワーク
- **特徴**: 超高速・軽量、マルチランタイム対応

### 技術的特徴

1. **パフォーマンス**

   - RegExpRouter による高速ルーティング
   - 14kB未満の軽量設計
   - Web標準APIのみ使用

2. **マルチランタイム対応**

   - Cloudflare Workers, Fastly, Deno, Bun, AWS Lambda, Node.js
   - 同じコードが全プラットフォームで動作

3. **開発者体験**

   - クリーンなAPI設計
   - ファーストクラス TypeScript サポート
   - 豊富なミドルウェアとヘルパー

4. **エコシステム**
   - 内蔵・カスタム・サードパーティミドルウェア対応
   - 包括的なヘルパー機能

## プロジェクト実行スケジュール

| イテレーション | 作業内容           | 期待スコア |
| -------------- | ------------------ | ---------- |
| 1              | 初期記事作成       | 6.0-7.0    |
| 2-3            | 基本的な改善       | 7.0-7.5    |
| 4-5            | 構成とフローの改善 | 7.5-8.0    |
| 6-7            | 技術的深度向上     | 8.0-8.3    |
| 8-9            | 人間らしさの向上   | 8.3-8.7    |
| 10             | 最終調整           | 8.5+       |

## 成果物

### 最終成果物

1. **高品質な技術記事** (Zenn投稿可能レベル)
2. **完成されたスタイルガイド** (今後の技術記事作成に活用可能)
3. **イテレーションログ** (各回のスコアと改善点の記録)

### 品質保証

- 技術的正確性の検証
- 実装コードの動作確認
- 読者視点での有用性検証

## 注意事項

1. **レビューの公正性**: Reviewerは厳格で建設的なフィードバックを提供する
2. **継続的改善**: 各イテレーションで明確な改善を目指す
3. **読者志向**: 最終的に読者にとって価値のある記事を目指す
4. **技術的正確性**: 実装例はすべて検証済みのものを使用する
